# Blueprints

Blueprints provide implementation guidance that complements the architectural contracts defined by the RFCs.

Unlike RFCs, Blueprints are informative rather than normative. They describe recommended implementation strategies, operational considerations, and future evolution without defining mandatory architectural contracts.

Relationship:

```
RFCs
    │
    │ define
    ▼
Architectural Contracts
    │
    │ guide
    ▼
Blueprints
    │
    │ support
    ▼
Implementation
```

Blueprints may evolve over time without changing the core architectural contracts.
